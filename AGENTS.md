- UI and layout feedback is often anchored to a **browser preview DOM selection**; expect iterative, viewport-aware tweaks. Profile-area pages (**Wallet**, **GoGoTrack**, **GoGoPass**, account sub-pages) use **`AccountPageShell`** on desktop — pass **`showProfileRail`** when pathname rail detection may fail; **`normalizePathname`** in the shell for profile-section routes — **profile sidebar rail** + centered **~878px** content column — not a standalone full-width **`phoneFrame`**.
