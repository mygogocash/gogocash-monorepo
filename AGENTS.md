## Learned User Preferences

- Verify admin ↔ customer changes on the **real API** (matching URL on both apps) — admin mock mode does not persist to the customer app.
- Stay on **MongoDB Atlas** for database infrastructure — do not plan GCP Cloud SQL/Firestore migration unless explicitly requested.
- UI and layout feedback is often anchored to a **browser preview DOM selection**; expect iterative, viewport-aware tweaks. Profile-area pages (**Wallet**, **GoGoTrack**, **GoGoPass**, account sub-pages) use **`AccountPageShell`** on desktop — **profile sidebar rail** + centered **~878px** content column — not a standalone full-width **`phoneFrame`**. Desktop footers share **`CustomerDesktopFooter`** via **`CustomerDesktopFooterSlot`** + **`getDesktopFooterHorizontalPadding()`** (full-bleed breakout). **FigJam/Figma SVG** imports (e.g. `docs/assets/gototrack-agentic-architecture.svg`) need **flat absolute coordinates** — no nested `transform` groups.
- Customer **auth**: web `/login` → Firebase phone OTP (`EXPO_PUBLIC_FIREBASE_*`, reCAPTCHA, authorized domains); **native Android/iOS** uses SecureStore **backend JWT** (Firebase phone OTP is **Expo web only** — do not debug native wallet/sign-in as Firebase OTP).
- Use **24-hour English time** in admin date/time pickers (flatpickr `HH:mm` + native `type=time`); do not show AM/PM.
- Surface **real API error messages** in admin forms/toasts via `getApiErrorMessage()` — avoid generic failures when the API returns a specific reason.
- Commit and push **only when explicitly asked**; keep diffs scoped to the task at hand.
- **Dark mode** (System / Light / Dark) is **customer app only** (`apps/app`); do not add tri-state theme work to admin as part of mobile tasks. Account sub-pages (e.g. Consent Preferences `/privacy-center`) use **`colors.card`** shell + **`colors.fieldMuted`** nested cards — not mint **`#F3FCF9`/`primarySoft`** page wash (match Account Settings).
- Do not wire **Crossmint**, **Customer.io**, or **Web3/ethers** flows under mobile `backend` mode — Firebase phone OTP + bank/PromptPay only.
- Involve Asia postback URL in the affiliate portal must be a **plain HTTPS URL** with `{macros}` — never paste a `curl` command or shell syntax into the postback field.
- Primary admin operational entry should stay **one click** from the sidebar (**Platform Dashboard**), not buried in a submenu.
- For mobile/Android and GoGoTrack work, prefer **Railway dev** (`api.dev.gogocash.co`) over staging. GoGoTrack **device acceptance** (Phase 5/7): owner supplies **`EXPO_TOKEN`**, **`GOGOTRACK_AUTH_TOKEN`** (backend JWT — on staging mint with Railway **`JWT_SECRET`** + seeded user), and a physical Android; run **`gototrack:preflight`** with **`--merchant-packages com.shopee.th --detect-package com.shopee.th`** for Shopee; re-grant **POST_NOTIFICATION** after Android OS updates; seed merchants via **Railway dev API shell** (local `railway run` cannot reach `mongo-staging.railway.internal`); agent may run preflight/`adb` but cannot grant Usage Access remotely. **Notification listener matching** is **not implemented** — Shopee MVP is **Usage Access + background cashback notifications** only; do not block on listener grant. Staging preflight: **`POST /gototrack/settings`** with **`enabled: true`** before detect or API returns **400** tracking disabled. For cross-app E2E, prefer **`npm run e2e`** or a manual local stack on the same API host.

## Learned Workspace Facts

- Monorepo apps: `apps/api` NestJS **:8080**, `apps/admin` Next.js **:3000**, `apps/app` Expo web **:8081**; dev Mongo **`gogocash-mongo`** on `:27017`; E2E harness uses Mongo RS **`gogocash-mongo-e2e`** + DB **`gogocash-e2e`**.
- **Local E2E:** manual stack — align `NEXT_PUBLIC_API_URL` / `EXPO_PUBLIC_API_URL` to `http://localhost:8080`, `npm run seed:local-admin` (`admin@gogocash.co` / `1234`). Automated — **`npm run e2e`** (`docker-compose.e2e.yml`, `.e2e/seed.json`, Playwright cross-system/admin/customer). Withdraw/money tests need replica set. Customer E2E uses seeded backend JWT (no Firebase OTP). **`E2E_DISABLE_RATE_LIMIT=1`** only when `NODE_ENV !== 'production'`.
- Without **`NEXT_PUBLIC_API_URL`**, admin uses in-memory **`/api/mock`** (offer ids like `o1`, `o2`) — changes do not reach the customer app or Mongo.
- **Hosted dev (Railway):** `https://api.dev.gogocash.co`, `https://admin.dev.gogocash.co` (admin login needs **`NEXTAUTH_SECRET`** + **`NEXTAUTH_URL=https://admin.dev.gogocash.co`**). **Hosted staging (Railway):** `https://api-staging.gogocash.co`, `https://admin-staging.gogocash.co` (hyphen domains; not `api.staging`). Staging **`MONGO_URI` without DB path → Mongoose default DB `test`** — seed **`test.gogosense_merchants`**, **`test.feerates`** (wallet **`POST /withdraw/check`**), or add `/gogocash` to URI. **Local UI + remote API:** point `EXPO_PUBLIC_API_URL` / `NEXT_PUBLIC_API_URL` at dev or staging — no local Mongo/API required. Customer wallet **"We could not load wallet"** = failed **`POST /withdraw/check`** (`useCustomerAccountResource`); common causes: stale **`demo-session`** in SecureStore (fixtures OTP), missing **`feerates`** doc, or user absent in Mongo; native API client prefers SecureStore JWT (Firebase ID token on web only). A valid withdraw/check with **`data: []`** is a zero-balance wallet — show the full dashboard, not a full-page empty state; coerce string amount fields if needed.
- **`Offer.commissions`** from the real API are Involve-style objects (`{ Commission: "5%" }`), not plain strings — parse in `apps/admin/src/lib/offerDeeplink.ts`.
- **Non-production API media:** when GCS auth fails, uploads fall back to `apps/api/.local-media/` with `local-media:` refs; admin `pathImage()` streams them via `/admin/stored-media/stream`.
- Local GCS uploads need **`GOOGLE_APPLICATION_CREDENTIALS`** as a **service-account JSON** — user ADC (`gcloud auth application-default login`) causes `invalid_grant` / `invalid_rapt`.
- **Railway vs Expo:** Railway project **GoGoCash** hosts **API + admin**; **staging customer web** is Railway **`@gogocash/mobile`** at **`https://app-staging.gogocash.co`** (CNAME → `*.up.railway.app`) — remove Cloudflare **`app-staging-proxy`** Worker or Firebase “Backend Not Found” persists. **Production** customer web remains legacy Next.js at **`app.gogocash.co`**; **dev** Expo web stays at **0 replicas**. **Production `api.gogocash.co` still routes to GCP Cloud Run**, not Railway. **Native** iOS/Android ship via **EAS Build**; **`dev` branch uses Expo SDK 57** — rebuild dev-client after SDK or native plugin bumps; `development` profile targets **`api.dev.gogocash.co`**; GitHub **`staging` environment** supplies `EXPO_PUBLIC_FIREBASE_*` (no separate `dev` GH env). `EXPO_PUBLIC_*` is inlined at EAS build — rebuild after API URL changes. **Preview/staging APK** JS-only UI fixes need **`eas update --channel staging --environment preview`** (or rebuild) — local code does not reach the device until published; preview builds need **`EXPO_PUBLIC_EAS_PROJECT_ID`** + **`updates.url`** at build time or OTA never applies — app **auto-reloads OTA on launch** when enabled. **Native Kotlin** changes (e.g. `GototrackMonitorService`) require **EAS rebuild** — OTA cannot ship them. **mongo-staging** on kernel **6.19+** needs **`mongo:8.0.4`** + start command exporting `GLIBC_TUNABLES=glibc.pthread.rseq=1` (do not use `rseq=0` — it overrides the service variable and triggers SERVER-121912); dev mongo can crash and need restart. Keep API **sleep off** and **1 replica** for in-process cron. Runbook: `docs/railway-execution-runbook.md`.
- **GoGoTrack (`dev` branch):** GoGoSense renamed to GoGoTrack on **`dev`** only (`main` still GoGoSense). Routes/code use `gototrack`; **Mongo collections remain `gogosense_*`** — merchant seed must upsert **`gogosense_merchants`** with **`brand_id`** (not `gototrack_merchants`). Customer entry: **Profile → GoGoTrack** (`profileHubMenuItems` → `/gototrack`); desktop profile rail **accordion submenu** (`profileHubGoGoTrackSubNavItems`: overview, setup, permissions, timeline, settings, recovery). Android MVP is **UsageStats + `GototrackMonitorService`** (`modules/gototrack-detector`) — **no `NotificationListenerService`**; settings UI marks **notification listener matching** unsupported. Railway dev **`gogocash-api`** needs **`INVOLVE_SECRET`** for `POST /gototrack/activate`. **Agent API (Phase 1):** optional **`merchantHint`** on detect, **`GET /gototrack/merchants/search`**, activation sources **`gototrack_background_prompt`** / **`gototrack_agent`**. Agentic copy: **AI NOTIFICATION**, **BRAND**, **3 STEPS SUMMARY** (BROWSE → ACTIVATE → EARN) — `docs/gototrack-agentic-architecture.md`. CI gates: `npm run test:gototrack` + `test:gototrack:api -w @gogocash/mobile`. Preflight defaults to **`api.dev.gogocash.co`**; auth via **`GOGOTRACK_AUTH_TOKEN`** with **`GOTOTRACK_AUTH_TOKEN` / `GOGOSENSE_AUTH_TOKEN` fallback**. **Phase 7:** background monitor needs **`GototrackMonitorService`** in the installed APK — **EAS dev-client/preview rebuild** before `--require-background-prompt` preflight can pass.
- **MongoDB Atlas:** prod cluster **`gogocash`** (M10); staging **`gogocash-staging`** (M0, **512 MB max**) — a full prod backup will not fit staging M0; use partial `mongorestore` or local Docker.
- **npm workspaces** hoist inconsistently — run `npm ci` at the monorepo root; if `-w` workspace commands fail module resolution, start from `apps/app` or `apps/admin`, or run the API with `NODE_PATH=./node_modules node dist/main`.
- Admin multipart uploads (FormData): strip default JSON `Content-Type` via **`AxiosHeaders.setContentType(false)`** so axios/browser sets the multipart boundary.
