## Learned User Preferences

- Verify admin тЖФ customer changes on the **real API** (matching URL on both apps) тАФ admin mock mode does not persist to the customer app.
- Stay on **MongoDB Atlas** for database infrastructure тАФ do not plan GCP Cloud SQL/Firestore migration unless explicitly requested.
- UI and layout feedback is often anchored to a **browser preview DOM selection**; expect iterative, viewport-aware tweaks.
- Customer **sign-in** issues on `/login` тЖТ investigate Firebase phone OTP (reCAPTCHA, authorized domains, `EXPO_PUBLIC_FIREBASE_*`), not admin staging credentials.
- Use **24-hour English time** in admin date/time pickers (flatpickr `HH:mm` + native `type=time`); do not show AM/PM.
- Surface **real API error messages** in admin forms/toasts via `getApiErrorMessage()` тАФ avoid generic failures when the API returns a specific reason.
- Commit and push **only when explicitly asked**; keep diffs scoped to the task at hand.
- **Dark mode** (System / Light / Dark) is **customer app only** (`apps/app`); do not add tri-state theme work to admin as part of mobile tasks.
- Do not wire **Crossmint**, **Customer.io**, or **Web3/ethers** flows under mobile `backend` mode тАФ Firebase phone OTP + bank/PromptPay only.
- Involve Asia postback URL in the affiliate portal must be a **plain HTTPS URL** with `{macros}` тАФ never paste a `curl` command or shell syntax into the postback field.
- Primary admin operational entry should stay **one click** from the sidebar (**Platform Dashboard**), not buried in a submenu.
- For mobile/Android and GoGoTrack work, prefer **Railway dev** (`api.dev.gogocash.co`) over staging. GoGoTrack **device acceptance** (Phase 5): owner supplies **`EXPO_TOKEN`**, **`GOGOTRACK_AUTH_TOKEN`** (customer JWT тАФ native Firebase phone OTP is **Expo web only**), and a physical Android; seed merchants via **Railway dev API shell** (local `railway run` cannot reach `mongo-staging.railway.internal`); agent may run `gototrack:preflight`/`adb` but cannot grant Usage Access remotely. For cross-app E2E, prefer **`npm run e2e`** or a manual local stack on the same API host.

## Learned Workspace Facts

- Monorepo apps: `apps/api` NestJS **:8080**, `apps/admin` Next.js **:3000**, `apps/app` Expo web **:8081**; dev Mongo **`gogocash-mongo`** on `:27017`; E2E harness uses Mongo RS **`gogocash-mongo-e2e`** + DB **`gogocash-e2e`**.
- **Local E2E:** manual stack тАФ align `NEXT_PUBLIC_API_URL` / `EXPO_PUBLIC_API_URL` to `http://localhost:8080`, `npm run seed:local-admin` (`admin@gogocash.co` / `1234`). Automated тАФ **`npm run e2e`** (`docker-compose.e2e.yml`, `.e2e/seed.json`, Playwright cross-system/admin/customer). Withdraw/money tests need replica set. Customer E2E uses seeded backend JWT (no Firebase OTP). **`E2E_DISABLE_RATE_LIMIT=1`** only when `NODE_ENV !== 'production'`.
- Without **`NEXT_PUBLIC_API_URL`**, admin uses in-memory **`/api/mock`** (offer ids like `o1`, `o2`) тАФ changes do not reach the customer app or Mongo.
- **Hosted dev (Railway):** `https://api.dev.gogocash.co`, `https://admin.dev.gogocash.co` (admin login needs **`NEXTAUTH_SECRET`** + **`NEXTAUTH_URL=https://admin.dev.gogocash.co`**). **Hosted staging:** `https://api-staging.gogocash.co`, `https://admin-staging.gogocash.co`. **Local UI + remote API:** point `EXPO_PUBLIC_API_URL` / `NEXT_PUBLIC_API_URL` at dev or staging тАФ no local Mongo/API required.
- **`Offer.commissions`** from the real API are Involve-style objects (`{ Commission: "5%" }`), not plain strings тАФ parse in `apps/admin/src/lib/offerDeeplink.ts`.
- **Non-production API media:** when GCS auth fails, uploads fall back to `apps/api/.local-media/` with `local-media:` refs; admin `pathImage()` streams them via `/admin/stored-media/stream`.
- Local GCS uploads need **`GOOGLE_APPLICATION_CREDENTIALS`** as a **service-account JSON** тАФ user ADC (`gcloud auth application-default login`) causes `invalid_grant` / `invalid_rapt`.
- **Railway vs Expo:** Railway project **GoGoCash** hosts **API + admin**; **`@gogocash/mobile` Expo web is at 0 replicas** (domains removed) тАФ customer web is legacy Next.js at **`app.gogocash.co`** (prod) and **`app-staging.gogocash.co`** (hyphen staging; not `app.staging`). **Production `api.gogocash.co` still routes to GCP Cloud Run**, not Railway. **Native** iOS/Android ship via **EAS Build**; `development` profile targets **`api.dev.gogocash.co`**; GitHub **`staging` environment** supplies `EXPO_PUBLIC_FIREBASE_*` (no separate `dev` GH env). `EXPO_PUBLIC_*` is inlined at EAS build тАФ rebuild after API URL changes. **mongo-staging** on kernel **6.19+** needs **`mongo:8.0.4`** + start command exporting `GLIBC_TUNABLES=glibc.pthread.rseq=1` (do not use `rseq=0` тАФ it overrides the service variable and triggers SERVER-121912); dev mongo can crash and need restart. Keep API **sleep off** and **1 replica** for in-process cron. Runbook: `docs/railway-execution-runbook.md`.
- **GoGoTrack (`dev` branch):** GoGoSense renamed to GoGoTrack on **`dev`** only (`main` still GoGoSense). Routes/code use `gototrack`; **Mongo collections remain `gogosense_*`** тАФ merchant seed must upsert **`gogosense_merchants`** with **`brand_id`** (not `gototrack_merchants`). Railway dev **`gogocash-api`** needs **`INVOLVE_SECRET`** for `POST /gototrack/activate`. CI gates: `npm run test:gototrack` + `test:gototrack:api -w @gogocash/mobile`. Preflight defaults to **`api.dev.gogocash.co`**; auth via **`GOGOTRACK_AUTH_TOKEN`** with **`GOTOTRACK_AUTH_TOKEN` / `GOGOSENSE_AUTH_TOKEN` fallback**.
- **MongoDB Atlas:** prod cluster **`gogocash`** (M10); staging **`gogocash-staging`** (M0, **512 MB max**) тАФ a full prod backup will not fit staging M0; use partial `mongorestore` or local Docker.
- **npm workspaces** hoist inconsistently тАФ run `npm ci` at the monorepo root; if `-w` workspace commands fail module resolution, start from `apps/app` or `apps/admin`, or run the API with `NODE_PATH=./node_modules node dist/main`.
- Admin multipart uploads (FormData): strip default JSON `Content-Type` via **`AxiosHeaders.setContentType(false)`** so axios/browser sets the multipart boundary.
