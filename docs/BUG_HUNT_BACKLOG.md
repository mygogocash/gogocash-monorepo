## How to work this in Cursor

1. Fix top-down by severity. Each item lists exact `file:line`, the failure scenario, and a suggested fix.
2. For every fix: add/adjust a test (the repo pins behavior heavily — `apps/app` uses vitest `vitest.config.ts` for unit + `vitest.render.config.ts` for render; `apps/api` uses jest).
3. Gate before commit: `npm run typecheck -w @gogocash/mobile`, `npm run test -w @gogocash/mobile`, `npm run test:render -w @gogocash/mobile`, and `npm run lint:ci -w gogocash-api` / `npm test -w gogocash-api` for API changes.
4. JS-only mobile fixes ship via staging OTA on push; native/config changes need EAS rebuild.

## The root-cause theme (read first)

Most P0/P1 mobile bugs are **one class**: server state held in per-component `useState` instead of a shared cache. Two mounted instances diverge; a mutation through one is invisible to the others until remount. We already hit this with the GoGoTrack settings toggle. **The highest-leverage fix is structural** — move these onto React Query (already a dependency) with shared query keys + `setQueryData` on mutation — which collapses findings #2, #3, #4, #5, #12 (background-prompts race) at once. Do that before or instead of patching them individually.

- **Location:** `apps/api/src/withdraw/withdraw.service.ts:1362`  ·  **Dimension:** api-money  ·  **Verify:** UNVERIFIED (reconfirm)
- **Evidence:** checkWithdrawMyCashback falls back to `this.userMyCashbackModel.find({ email: { $regex: user.email } })` (line 1358-1365). ### 8. [P2] Auto-created MyCashback withdrawal is deducted from no balance when mycashback_id is absent (repeatable MCB drain)

- **Location:** `apps/api/src/withdraw/withdraw.service.ts:1736`  ·  **Dimension:** api-money  ·  **Verify:** UNVERIFIED (reconfirm)
- **Evidence:** In create() (and identically createBankTransfer at 2060-2079) the server auto-creates a second withdraw record for `availableMCB` but tags it `mycashback_id: createWithdrawDto.mycashback_id ? map(...) : undefined` (lines 1736-1738). checkWithdraw only counts conversion withdrawals via `mycashback_id: []` (line 393-397) and checkWithdrawMyCashback only deducts via `mycashback_id: { $in: [...] }` (line 1421-1423).
