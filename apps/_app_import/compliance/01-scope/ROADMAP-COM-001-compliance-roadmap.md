## Purpose

High-level **time-bound plan** for IMS maturity: **SOC 2 Type II**, **ISO 9001**, **ISO 27001:2022**, sized for a **~5-person** team.

## Roadmap summary

| Horizon | Focus | Outcomes |
| --- | --- | --- |
| **0–30 days** | Scope, ownership, inventories, framework, risk methodology, policy drafting, repo structure | Baseline docs, risk register v1, control matrix v1 |
| **31–90 days** | MFA, logging, access reviews, backups, IR drills, training, vendor reviews, **1st internal audit**, **1st MR** | Working technical controls; first audit cycle |
| **91–180 days** | Recurring evidence, KPI review, CAPA, restore tests, **2nd internal audit** | Trend data; closed gaps |
| **6–12 months** | Quarterly reviews, SOC 2 evidence quality, supplier reviews, **SOC 2 Type II window** / ISO readiness | Audit-ready evidence set |

**Detail:** See `compliance/IMPLEMENTATION_ROADMAP_30_90_180_365.md` (companion file — generated in same package).

## Procedure / operating rules

1. **Roadmap** is updated in **management review** or when major scope changes.  
2. **Slippage** >30 days on critical controls (e.g., MFA, backups) requires **documented** reason and new target date in risk or CAPA.
